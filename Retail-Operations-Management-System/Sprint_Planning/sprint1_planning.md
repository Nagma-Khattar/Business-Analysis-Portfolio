# Sprint 1 – Core Retail Operations

## Sprint Goal

Deliver the core retail functionality required for secure user access, inventory management, and sales transactions.

## Sprint Duration

2 Weeks

## Selected User Stories

- ROMS-6 User Login Functionality
- ROMS-7 Role-Based Access Control
- ROMS-8 Add New Product
- ROMS-9 View Inventory
- ROMS-10 Update Inventory
- ROMS-11 Record Sales Transaction

## Deferred Stories

The following stories were moved to future sprints:

- ROMS-12 View Sales Report
- ROMS-13 Create Customer Profile
- ROMS-14 Generate Business Reports

## Prioritization Reason

Sprint 1 focuses on implementing the minimum functionality required for retail operations. Authentication, inventory management, and sales processing are foundational features. Reporting and customer management depend on operational data generated by these core capabilities and will be implemented in future sprints.
