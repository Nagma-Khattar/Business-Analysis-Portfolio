# Sprint 1 Retrospective

## Sprint Goal
Deliver core retail functionality for user access, inventory management, and sales transactions.

## What Went Well
- User stories were clearly defined.
- Acceptance criteria helped validate requirements.
- Jira workflow provided visibility.

## Challenges
- Story estimates were not defined before sprint start.
- Additional dependency analysis can improve planning.

## Action Items
- Add story point estimates in future sprints.
- Review acceptance criteria before sprint planning.
- Identify technical dependencies early.
